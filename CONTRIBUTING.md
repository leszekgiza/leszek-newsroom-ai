# Contributing

Thanks for your interest in contributing.

## Basics
- Open an issue before large changes.
- Keep PRs small and focused.
- Do not include API keys or credentials in code or logs.

## Development
- Node.js 20+
- PostgreSQL 15+
- Crawl4AI service (see README)

## Tests
- `npm run lint`
- `npm run test:e2e` (when relevant)

## Commit Style
- Short, descriptive messages
- One logical change per commit
